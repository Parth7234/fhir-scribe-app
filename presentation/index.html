The Problem

Context

  • Indian doctors spend 30-40% of time on manual documentation
  • Most clinical conversations happen in Hinglish — no tools support it
  • Paper prescriptions = no structured data for HMIS systems
  • Tier 2/3 city clinics lack digital infrastructure

Objective: Capture doctor-patient conversations in real-time (Hindi + English mix), convert them into structured clinical notes, and map to FHIR R4 compliant resources.

Our Solution

A mobile-first AI-powered clinical scribe that listens, transcribes, structures, and validates — all in real-time.

Hinglish Voice → Text

Real-time transcription with Doctor/Patient speaker tags using Gemini 2.5 Flash. Supports Hindi, English & Hinglish.

🚀 Deployment & Security

Production Deployment

🌐 Frontend — Vercel

React 19 + Vite production build, global CDN, instant deployments, environment variable management for API URLs

⚙️ Backend — Render

FastAPI + Uvicorn with CORS configuration, auto-deploy from Git, environment secrets for Gemini API key

📡 API Endpoints

POST /api/transcribe/ — Audio → Text
POST /api/fhir/ — Transcript → FHIR + Notes
POST /api/fhir/validate — Bundle Validation

Security & Authentication

🔐 Clerk Authentication

JWT-based secure doctor sign-in. JWKS public key verification on backend. Protected API routes — no unauthenticated access.

🛡️ Backend Route Protection

Every API endpoint requires Bearer token. Token signature verified against Clerk's JWKS endpoint using RS256 algorithm.

🔒 Frontend Protection

SignedIn/SignedOut guards. Axios interceptor auto-attaches auth tokens. Unauthorized users see login-only screen.
